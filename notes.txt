Kafka has high throughput but less storage in comparison of DB which has low throughput but huge storage
throughput means number of entries per second

Kafka has number of terminologies like : 
 - Consumer these consume the data that is produced also this can be client or server
 - Producer from here the data is produced , it can be client, or server
 - Topics these are the logical partitioning of messages also you can make partitions inside these topics as well

Architecture : 
- When producer produces a message it defines that this message is for which topic
